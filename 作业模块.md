# Prompt

你现在是一个资深 Windows 桌面应用架构师和高级 C# 工程师。请为我生成一个**完整可运行**的 Windows 作业批注打印软件项目，严格按下面要求实现。

---

## 一、项目目标

开发一个面向孩子做作业的 Windows 桌面程序，核心用户旅程如下：

1. 打开软件
2. 进入“作业”
3. 进入“学校作业”
4. 进入“语文”
5. 拖入老师发来的作业图片或 PDF
6. 点击“开始做作业”
7. 软件自动以合适缩放显示图片/PDF
8. 孩子直接在页面上手写作业
9. 可切换多页作业
10. 做完后点击“一键打印”
11. 软件直接打印当前作业

这个软件不是通用笔记软件，不要做成 OneNote，也不要做成白板。  
它是一个**作业专用器**，目标只有四件事：

- 导入图片/PDF
- 直接手写
- 多页切换
- 一键打印

---

## 二、最高优先级约束

### 1. 手写体验优先
手写体验不能差，目标是不弱于 OneNote 的基础书写体验。

要求：

- 使用 **Windows 原生 Ink 能力**
- 不要自己手写实现底层笔迹采样和渲染算法
- 必须使用适合 Windows 手写输入的方案
- 要支持压感（如果系统和设备支持）
- 页面上应当是“底图 + 墨迹层”的分层结构，不能把墨迹直接画死到底图上

### 2. 性能优先
这个软件必须重视性能，尤其是：

- 打开图片/PDF 不能卡
- 翻页不能明显卡顿
- 手写不能因为文档层卡住
- 打印前不能做非常重的阻塞操作

### 3. 范围收敛
第一版只做这些功能：

- 导入 PDF
- 导入 JPG / PNG / BMP
- 多页图片
- 多页 PDF
- 手写
- 橡皮
- 上一页 / 下一页
- 自动保存
- 一键打印
- 历史作业恢复

第一版**不要实现**以下功能：

- OCR
- AI 批改
- 云同步
- 富文本
- 图形工具
- 多种笔刷
- 文本框
- 账户系统
- 在线协作
- 网络依赖

## 五、架构要求

请按下面分层实现。

### 1. UI 层
页面建议至少包括：

- `MainWindow`
- 首页：作业入口
- 学校作业页
- 学科页（至少先放语文）
- 导入页
- 作业编辑页
- 历史作业页
- 设置页（至少打印机设置）

### 2. 文档层
定义统一文档接口，例如：

- 打开文档
- 获取页数
- 获取页面逻辑大小
- 渲染某一页

要求：

- PDF 走 `PdfDocumentProvider`
- 图片走 `ImageDocumentProvider`
- 多张图片视为多页文档
- 不要把图片先转成 PDF

### 3. 手写层
每页都要有独立手写层。

要求：

- 使用 `InkCanvas`
- 支持笔和触摸
- 支持橡皮
- 支持清空当前页
- 每页墨迹独立保存
- 翻页时切换当前页墨迹
- 墨迹存储不能绑死在屏幕像素坐标上，要基于页面逻辑坐标

### 4. 会话层
要有“作业会话”概念。

每次导入后创建一个 JobSession，至少包含：

- JobId
- Subject
- 来源文件列表
- 页数
- 当前页
- 创建时间
- 更新时间
- 是否打印过
- 每页墨迹文件路径

### 5. 打印层
要求：

- 孩子在编辑页点击“一键打印”
- 自动保存当前页墨迹
- 遍历作业所有页
- 将每页底图与墨迹层合成
- 直接发送到默认打印机

注意：

- 不要要求孩子每次复杂设置打印参数
- 设置页里可以配置默认打印机、纸张等
- 打印前可以弹一个很轻的确认框，但不能复杂

## 六、性能实现要求

请务必按下面要求实现：

### 1. 页面缓存
只缓存：

- 当前页
- 上一页
- 下一页

不要全量渲染整份文档。

### 2. 异步渲染
PDF 页面渲染必须放后台线程。  
不要在 UI 线程做重渲染。

### 3. 自动保存防抖
每次墨迹变化后，不要立刻频繁写盘。  
请实现防抖保存，例如：

- 笔迹变化后延迟几百毫秒再保存
- 翻页前强制保存
- 打印前强制保存
- 窗口关闭前强制保存

### 4. 底图和墨迹分层
必须是：

- 底层：文档页位图
- 顶层：InkCanvas

不要把墨迹直接烙死到底图位图。

### 5. 翻页优化
翻页时不要销毁整个编辑器。  
尽量复用编辑页控件，只切换：

- 当前页面底图
- 当前页墨迹

## 七、数据与文件结构要求

请使用本地文件夹存储作业数据。  
目录结构建议类似：

AppData/Jobs/{jobId}/
  job.json
  source/
  ink/
  cache/

其中：

- `job.json`：作业元数据
- `source/`：导入的原始 PDF 或图片
- `ink/`：每页墨迹文件
- `cache/`：可删的渲染缓存

要求：

- 历史作业可恢复
- 作业下次打开时能恢复到上次页码和笔迹
- 原始文件保留
- 缓存可删
